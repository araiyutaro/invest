## Spacing Scale

**注記:** 本プロジェクトは20フェーズにわたり確立済みのrem基準スケール（16pxルート想定）を採用しており、strict 4pxグリッドではない。Phase 20は新規スケールを導入せず、既存の値のみを再利用する。

| Token | Value (rem / px換算) | Usage |
|-------|----------------------|-------|
| badge-y | 0.15rem (2.4px) | 「社名一致」バッジの縦padding（`.ticker-pill` の既存値を踏襲） |
| badge-x | 0.4rem (6.4px) | 「社名一致」バッジの横padding・左マージン |
| item-y | 0.4rem (6.4px) | ニュース1件あたりの`<li>`縦padding、見出しmargin-bottom |
| section-gap | 0.8rem (12.8px) | 「関連ニュース」サブセクションのrationale/riskNoteからの上マージン |

Exceptions: 上記はすべてプロジェクト既存のrem値（0.15rem, 0.4rem, 0.8rem）からの再利用であり、4pxの厳密倍数ではない（レガシー一貫性を優先し新グリッドは導入しない — CONTEXT.md D-02の「モバイル表示と統一感のためリストを維持」方針に整合）。

## Typography

本フェーズで新規に組み込む要素のみを規定する（h1〜h3等の既存見出し階層は変更対象外）。

| Role | Size | Weight | Line Height |

| サブセクション見出し（「関連ニュース」） | 0.85rem (13.6px) | 600 (semibold) | 1.4 |

| Element | Copy |
|---------|------|
| Primary CTA | 該当なし — 本フェーズはインタラクティブなボタン/フォームを持たない静的レポート。唯一のインタラクション要素は元記事への外部リンク（`target="_blank" rel="noopener noreferrer"`） |

| マッチ方式バッジ | `社名一致`（`matchType === "name"` または `"alias"` の記事にのみ付与。`matchType === "ticker"` はバッジなし＝無印がデフォルト。CONTEXT.md D-07） |
| Empty state heading | `関連ニュース`（見出し自体を省略しない。上記と同一） |
| Empty state body | `本日の関連ニュースなし`（ミュートグレー`#888`、1行。CONTEXT.md D-08） |
| Error state | 該当するユーザー向け文言なし — fail-soft設計（CONTEXT.md D-09）。`tmp/holding-news.json` 欠損/パース失敗時は「0件」と完全に同一の空状態表示（`本日の関連ニュースなし`）に自然縮退させる。内部ログは `console.error` のみでユーザーには表示しない |
| ID解決失敗時の扱い | `tmp/news.json` に存在しないIDは該当エントリのみ静かにスキップ（ユーザー向け文言なし、ログのみ。CONTEXT.md D-10） |

## Component / Markup Contract（実装ノート）

executor向けの具体的なHTML構造契約。`formatHoldingEvaluationsHtml()` 内、各カードの `rationale`/`riskNote` の下に追加する。

**0件でない場合:**
```html
<div style="margin-top:0.8rem;">
  <p style="font-size:0.85rem;font-weight:600;color:#a5b4fc;margin-bottom:0.4rem;">関連ニュース</p>
  <ul style="list-style:none;padding-left:0;margin:0;">
    <li style="padding:0.4rem 0;border-top:1px solid #2a2a3e;background:transparent;border-radius:0;margin-bottom:0;">
      <a href="{escapeHtml(url)}" target="_blank" rel="noopener noreferrer">{escapeHtml(title)}</a>
      <!-- matchType が "name" | "alias" の場合のみ以下を出力 -->
      <span style="display:inline-block;background:#2a2a3e;color:#9ca3af;font-size:0.7rem;font-weight:500;padding:0.15rem 0.4rem;margin-left:0.4rem;border-radius:999px;">社名一致</span>
      <p style="color:#888;font-size:0.85rem;margin:0.15rem 0 0;">{escapeHtml(source)} ・ {formatPublishedAtJst(publishedAt)}</p>
    </li>
    <!-- 最大5件、holding-news.jsonの供給順をそのまま維持（並べ替え・絞り込み禁止 — CONTEXT.md D-04/D-05） -->
  </ul>
</div>
```

**0件の場合（欠損・パース失敗時も同一表示 — D-08/D-09）:**
```html
<div style="margin-top:0.8rem;">
  <p style="font-size:0.85rem;font-weight:600;color:#a5b4fc;margin-bottom:0.4rem;">関連ニュース</p>
  <p style="color:#888;font-size:0.85rem;">本日の関連ニュースなし</p>
</div>
```

**重要な実装制約（グローバルCSS衝突回避）:**
1. カードdivのclassは `class="agent-card news-card"` の2クラス併用にすること（リンク色自動適用のため。上記Color節参照）。
2. サブセクション見出しは `<h4>` ではなく `<p>` タグを使うこと — `.agent-card h4 { color: #a5b4fc; margin-top: 0; }` という既存グローバルルールが、カード先頭の銘柄名見出し（`<h4>${symbol}...`）と衝突し、意図しない上マージン相殺・色重複を起こすため。
3. `<li>` は必ずインラインstyleで `background:transparent; border-radius:0; padding:0.4rem 0; margin-bottom:0;` を明示指定すること — グローバル `li { background:#1e1e2e; border-radius:6px; border-left:3px solid ${accentColor}; padding:0.5rem 0.8rem; }` ルールを打ち消さないと、`#10b981`アクセントの`border-left`がニュース項目ごとに繰り返し表示され、Accent 10%予算を大幅超過する。
4. `<ul>` は必ず `margin:0` を明示指定すること — グローバル `ul { margin-bottom: 1rem; }` により、カード最後の要素になった場合に余分な下部余白が生じるため。
5. すべてのテキスト（title / source / バッジ文言）に `escapeHtml()` を適用すること。
6. `href` は `resolvePortfolioHoldingNews()` の照合結果から取得したURLのみを使用し、LLM出力由来のURLフィールドは存在しても無視すること（幻覚URL防止の最終ガード）。
