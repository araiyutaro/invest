<objective>
フィルタ済み記事プール（20〜80件）の各記事へ短い連番 ID（n01…n80）を付与する純関数 `assignArticleIds()` を実装し、`collect-data.ts` の `tmp/news.json` 書き出し直前に組み込む。これにより Phase 16/17 は「ID 付き news.json」を前提にでき、キュレーション Agent は短い ID をコピーするだけで記事を参照できる（D-01/D-02、CURA-02 のID参照方式の基盤）。

Purpose: LLM に URL/タイトルを自由生成させず、記事参照を短い連番 ID に限定することで幻覚URLを構造的に防止する土台を作る。
Output: `src/data/news/article-id.ts`（純関数）、`collect-data.ts` への統合、両者の単体テスト。
</objective>
