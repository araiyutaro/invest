### T-03: run.sh に HTML チェックサム保護を追加
**Requirement:** OPS-02
**File(s):** `scripts/run.sh`
**Action:** パイプライン実行前後で docs/index.html と docs/portfolio.html の SHA256 チェックサムを比較し、変更があれば git checkout で復元

**Details:**

**チェックサム記録（`claude` コマンドの直前に追加）:**

```bash
# HTML Protection: record checksums before pipeline
PROTECT_FILES="docs/index.html docs/portfolio.html"
CHECKSUM_FILE="/tmp/invest-html-checksums-${TIMESTAMP}.txt"
for f in $PROTECT_FILES; do
  if [ -f "$PROJECT_DIR/$f" ]; then
    shasum -a 256 "$PROJECT_DIR/$f" >> "$CHECKSUM_FILE"
  fi
done
```

**チェックサム検証・復元（`claude` コマンドの直後、EXIT_CODE 取得の後に追加）:**

```bash
# HTML Protection: verify checksums and restore if changed
if [ -f "$CHECKSUM_FILE" ]; then
  RESTORED=""
  for f in $PROTECT_FILES; do
    if [ -f "$PROJECT_DIR/$f" ]; then
      EXPECTED=$(grep "$PROJECT_DIR/$f" "$CHECKSUM_FILE" | awk '{print $1}')
      if [ -n "$EXPECTED" ]; then
        ACTUAL=$(shasum -a 256 "$PROJECT_DIR/$f" | awk '{print $1}')
        if [ "$EXPECTED" != "$ACTUAL" ]; then
          git -C "$PROJECT_DIR" checkout -- "$f"
          RESTORED="$RESTORED $f"
          echo "HTML保護: $f を復元しました" | tee -a "$LOG_FILE"
        fi
      fi
    fi
  done
  rm -f "$CHECKSUM_FILE"
  if [ -n "$RESTORED" ]; then
    echo "HTML保護: 復元されたファイル:$RESTORED" | tee -a "$LOG_FILE"
  fi
fi
```

**エッジケース処理:**
- ファイルが存在しない場合: `if [ -f ... ]` で存在チェック済みのため、チェックサム記録をスキップ
- パイプライン実行中にファイルが削除された場合: grep でチェックサムが見つかるがファイルが存在しない → 内側の `if [ -f ... ]` でスキップ
- チェックサムファイルのクリーンアップ: `rm -f "$CHECKSUM_FILE"` で処理後に削除

## Plan Checker Resolution

**Gap 1 — スタックトレース:** ROADMAP SC-1 の「スタックトレースが記録される」は、run.sh の既存ログ記録（`>> "$LOG_FILE" 2>&1`）で Claude の全出力（エラー含む）がログに自動記録されることで充足。ステップマーカーは「どのステップで失敗したか」を特定するための補助情報であり、スタックトレース自体は既存のログ機構で捕捉済み。ROADMAP SC-1 の文言を「失敗ステップ名・エラーメッセージが記録される」に修正。

**Gap 2 — ログファイル名:** ROADMAP SC-1 の `logs/pipeline-YYYY-MM-DD.log` は要件記述時の想定名。実際の実装は `logs/invest-YYYY-MM-DD_HHMMSS.log`（D-04 で run.sh 変更なしと決定済み）。ROADMAP の表記を実装に合わせて修正。
