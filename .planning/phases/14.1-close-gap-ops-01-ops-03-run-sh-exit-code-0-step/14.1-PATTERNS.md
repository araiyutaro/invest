### `scripts/run.sh` — `EXIT_CODE` capture (OPS-03, D-04)

**Current code** (`scripts/run.sh` lines 35-41):
```bash
claude --dangerously-skip-permissions \
  -p "/invest" \
  --model claude-sonnet-4-6 \
  --max-turns 200 \
  >> "$LOG_FILE" 2>&1 || true

EXIT_CODE=${PIPESTATUS[0]:-$?}
```

**No in-repo analog** — self-contained bugfix. Root cause per audit (`.planning/v2.3-MILESTONE-AUDIT.md` line 77): `|| true` unconditionally overwrites `PIPESTATUS[0]` to 0, so `EXIT_CODE` is always 0 regardless of the actual `claude` exit status. Constraint from D-04: the script runs under `set -euo pipefail` (line 2), so simply removing `|| true` would abort the script immediately on non-zero exit — skipping the HTML-protection restore, notification, and log-rotation steps below. The fix must capture the real exit code **without** triggering early script termination (Claude's Discretion: `set +e`/`if` branch or equivalent, e.g. temporarily disabling `errexit` around the `claude` call, or wrapping the call in an `if ... ; then ... else ... fi` so the non-zero status is consumed by the conditional rather than propagating).

### `scripts/run.sh` — checksum lookup `grep` fix (WR-01)

**Analog:** `.planning/phases/13-operational-stability/13-REVIEW.md` lines 65-75 (fix fully specified, ready to copy)

**Current code** (`scripts/run.sh` line 48):
```bash
EXPECTED=$(grep "$PROJECT_DIR/$f" "$CHECKSUM_FILE" | awk '{print $1}')
```

**Fix to copy verbatim** (13-REVIEW.md line 74):
```bash
EXPECTED=$(grep -F "$PROJECT_DIR/$f" "$CHECKSUM_FILE" | awk '{print $1}')
```

**Why:** unquoted `grep` interprets the path as a regex; any `.` in the path matches any character, risking a wrong checksum-line match. `-F` forces literal/fixed-string matching.
