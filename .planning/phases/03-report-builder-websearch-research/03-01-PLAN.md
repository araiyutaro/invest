<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebSearch → Agent subagent | 外部ウェブ検索結果がサブエージェント内に取り込まれる |
| WebFetch → Agent subagent | 外部記事コンテンツがサブエージェント内に取り込まれる |
| Agent subagent → tmp/websearch/*.json | WebSearch 結果の JSON がファイルに保存される |
| Agent subagent → tmp/reeval/*.json | 再評価結果の JSON がファイルに保存される |
| tmp/*.json → generate-report.ts | 中間ファイルが TS スクリプトに読み込まれる |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | WebSearch Agent JSON output | mitigate | Zod スキーマ (webSearchResultSchema) でバリデーション。不正 JSON 時はフォールバックオブジェクトを保存 |
| T-03-02 | Tampering | Reevaluation Agent JSON output | mitigate | Zod スキーマ (reevaluationOutputSchema) でバリデーション。不正 JSON 時はフォールバックオブジェクト |
| T-03-03 | Spoofing | WebSearch/WebFetch results | accept | 外部ウェブコンテンツは信頼できないが、投資分析の参考情報としての使用に限定。最終判断はユーザー |
| T-03-04 | Information Disclosure | tmp/websearch/*.json, tmp/reeval/*.json | accept | Phase 1 の .gitignore で tmp/ がgit管理外。個人利用・ローカル環境限定 |
| T-03-05 | Denial of Service | 大量の highlightedStocks | accept | 実際のミーティングで生成される銘柄数は通常5-15件。並列 Agent 数は実用的な範囲 |
| T-03-SC | Tampering | npm installs | accept | 新規パッケージインストールなし |
</threat_model>
