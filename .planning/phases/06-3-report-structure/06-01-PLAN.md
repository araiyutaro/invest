## Goal

現行の単一 `generate-report.ts`（478行）を複数モジュールに分離し、`docs/YYYY-MM-DD/` に `daily-report.html`、`meeting-minutes.html`、`portfolio-report.html` の3ファイルを出力する基盤を構築する。共通ユーティリティの抽出、Round 1/2/3 データローダーの追加、Daily Report の既存 HTML 生成関数の移動、`invest.md` Step 3c の更新を行う。Meeting Minutes・Portfolio Report の HTML テンプレート実装は Plan 06-02 で行う（本プランではスタブ関数で仮出力）。

## Architecture Decision

**D-01 判断: 3ファイル+共通モジュール方式を採用**

```
src/scripts/
├── report-utils.ts              # 共通ユーティリティ（escapeHtml, markdownToHtml, CSS, color helpers）
├── report-data-loaders.ts       # Round 1/2/3 データローダー
├── generate-daily-report.ts     # Daily Report HTML 生成
├── generate-meeting-minutes.ts  # Meeting Minutes HTML 生成（06-02で実装、本プランではスタブ）
├── generate-portfolio-report.ts # Portfolio Report HTML 生成（06-02で実装、本プランではスタブ）
├── generate-report.ts           # オーケストレーター（エントリーポイント維持）
└── generate-report.test.ts      # テスト
```

理由: 現行478行に Meeting Minutes の全アナリスト表示を加えると800行超になるため、ユーザーのコーディング規約（200-400行目安、800行上限）に従い分離する。`generate-report.ts` は `invest.md` Step 3c から `npx tsx src/scripts/generate-report.ts` で呼ばれるため、エントリーポイントとして維持する。
