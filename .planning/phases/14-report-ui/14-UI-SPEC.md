# Phase 14 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** This project is NOT a React/Next/Vite SPA. HTML reports are generated server-side as
TypeScript template literals (`src/scripts/generate-*.ts`, shared styles in `src/scripts/report-utils.ts`)
and written to static files under `docs/`. There is no component framework, no bundler, no npm UI
dependencies. The shadcn gate is therefore **not applicable** — `Tool: none`. All contract values below
are plain CSS declared inside `<style>` blocks / inline SVG attributes.

| Property | Value |

## Spacing Scale

Existing codebase uses ad-hoc rem values (0.3rem–3rem) rather than a strict 4px grid. Per CONTEXT.md D-03 the existing theme must be preserved, so this contract does **not** retrofit existing rules. All **new** elements built in this phase (hero section, accordion, chart containers, mobile breakpoint adjustments) MUST follow the 8-point scale below.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Chart tick/label gaps, SVG padding |
| sm | 8px | Compact spacing inside accordion summary, chart legend items |
| md | 16px | Default spacing: hero section internal padding, chart container margin |
| lg | 24px | Section padding between hero and archive accordion |
| xl | 32px | Gap between major page sections on desktop |
| 2xl | 48px | Not used in this phase (reserved) |
| 3xl | 64px | Not used in this phase (reserved) |

Exceptions:
- Existing report body content (`report-utils.ts`) keeps its current rem-based spacing (0.5rem/0.8rem/1rem/1.2rem/1.5rem/2rem/3rem) — out of scope for retrofit, do not change.
- Tap target minimum: 44px height for all interactive elements (links, `<summary>` accordion headers) at ≤768px viewport, per D-10. This is a WCAG/mobile-usability exception to the spacing grid, not a spacing token.

## Typography

Existing report body typography (h1 1.8rem/28.8px bold, h2 1.4rem/22.4px, h3 1.15rem/18.4px, h4 1rem/16px, body 16px/1.7, meta text 0.75–0.9rem) is established and preserved per D-03 — do not re-declare or change these for existing report content.

For **new** elements introduced in this phase (index.html hero section, month accordion, SVG chart labels/legends/axis text), declare:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.7 (matches existing) |
| Label | 13px | 400 (regular) | 1.4 |
| Heading | 20px | 700 (bold) | 1.3 |
| Display | 28px | 700 (bold) | 1.2 |

Weight declaration: exactly 2 weights — **400 regular** (body copy, chart axis labels) and **700 bold** (headings, hero date, accordion month labels, `<strong>` emphasis — matches existing `font-weight: bold` usage, no semibold introduced to avoid a third weight).

Usage mapping:
- Display (28px/700) → index.html hero section headline (latest report date/title)
- Heading (20px/700) → accordion `<summary>` month labels (e.g. "2026年7月"), chart section titles ("VIX推移", "セクターパフォーマンス")
- Body (16px/400) → hero section summary text, report link labels
- Label (13px/400) → SVG chart axis ticks, legend text, threshold line annotations (consolidates existing 0.75–0.85rem meta-text range into one token for new chart UI)

Mobile scale-down (≤768px, per D-11): Display → 22px, Heading → 18px. Body and Label sizes remain unchanged (already legible at 375px).

## Color

Existing per-report accent theming is preserved (D-03): Daily Report / index.html = `#3b82f6` (blue), Meeting Minutes = `#f59e0b` (amber), Portfolio Report = `#10b981` (green). This is an established pattern, not new.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f0f1a` | Page background (all HTML files — existing) |
| Secondary (30%) | `#1e1e2e` (cards/tables), `#1a1a28` (discussion cards), `#2a2a3e` (table header row) | Cards, table backgrounds, accordion panel background — existing |
| Accent (10%) | `#3b82f6` (index.html / daily report scope for Phase 14) | Reserved for: hero section border/headline accent, accordion `<summary>` marker color, active/hover link states, VIX chart line stroke |
| Destructive/Negative | `#ef4444` | Reserved for: negative sector bars (D-05), bearish score coloring (existing `scoreColor`/`verdictColor` — reused, not a UI action) |
| Positive (semantic, new) | `#10b981` | Reserved for: positive sector bars (D-05) |
| Threshold marker (new) | `#6b7280` (gray, dashed) | VIX chart 20/30 threshold reference lines — neutral, not accent, not semantic pos/neg |

Accent reserved for: hero headline border-bottom / accent bar, accordion `<summary>` marker + hover state, VIX chart line stroke and its data-point dots. Do NOT use accent color for sector bars (those use the positive/negative semantic pair per D-05) or for body text.

Note on "Destructive": this project has no delete/mutate user actions (static generated reports, no forms). `#ef4444` is retained here as the negative/bearish semantic color already established by `scoreColor()`/`verdictColor()` in `report-utils.ts`, extended to sector-bar negative values per D-05.

## Copywriting Contract

This phase has no user-input forms and no destructive actions (static, auto-generated report site). Contract covers report-viewer copy only.

| Element | Copy |
|---------|------|
| Primary CTA (existing, preserved) | Report links keep existing labels: "Daily Report", "Meeting Minutes", "Portfolio Report" |
| Accordion summary label (new) | `{YYYY年M月} ({N}件)` — e.g. "2026年7月 (3件)" |
| Chart empty/unavailable state heading | "データ取得エラー" |
| Chart empty/unavailable state body | "{VIX推移 / セクターパフォーマンス}データを取得できませんでした。次回のレポート生成をお待ちください。" — shown as plain text in place of the SVG when `vixHistory` or `sectors` array is empty/missing, no retry action available (batch-generated site) |
| Archive empty state (no reports in a month group) | Not applicable — accordions are only rendered for months that have ≥1 report entry |
| Error state (general page-level fetch failure) | Not applicable — pages are static pre-rendered HTML, no client-side fetch/error path exists |
| Destructive confirmation | None — no destructive actions exist in this phase |

## Chart Contracts (SVG, inline-generated)

Per D-04/D-05/D-06, charts are server-generated inline SVG strings embedded directly in the HTML template output (no client-side JS, no external chart library).

### Sector Performance (horizontal bar chart)
- Data source: `market.json` → `sectors[].changePercent` (11 sectors)
- Sort order: descending by `changePercent` (largest positive at top) per Specific Ideas note
- Bar color: `#10b981` for `changePercent >= 0`, `#ef4444` for `changePercent < 0`
- Bar height: 24px (spacing `md`-adjacent), gap between bars: 8px (`sm`)
- Label text: sector name + `changePercent` value, 13px/400, color `#e0e0e0` (existing body text color)
- Container: full-width within `.container` (max 960px per existing), horizontally scroll-free at 375px (bars scale to container width, not fixed px width)

### VIX History (line chart)
- Data source: `market.json` → `vixHistory: [{date, close}]`, 30 daily points (D-07/D-08)
- Line stroke: accent `#3b82f6`, stroke-width 2px
- Data point dots: 3px radius, same accent color
- Threshold reference lines: dashed, `#6b7280`, at y=20 and y=30, with 13px/400 gray label ("20", "30") at the line end
- Axis labels: 13px/400, color `#888` (existing meta-text gray)
- Container: `viewBox`-based SVG (e.g. `viewBox="0 0 600 200"`) so it scales fluidly to any container width including 375px — no fixed pixel width on the `<svg>` element itself, only on `viewBox`

## Mobile Responsive Contract

Per D-09/D-10/D-11, applies to all 5 HTML surfaces: `index.html`, `docs/{date}/daily-report.html`, `meeting-minutes.html`, `portfolio-report.html`, `portfolio.html`.

| Breakpoint | Rule |
|------------|------|
| `@media (max-width: 768px)` | `body` padding 2rem → 1rem; Display/Heading font sizes reduced per Typography section; `.container` max-width becomes `100%` |
| 375px (iPhone SE, tested at this width, not a separate media query — result of the 768px rules) | Tables: wrap in `overflow-x: auto` container OR switch to stacked `display: block` row layout (implementer's discretion per CONTEXT.md Claude's Discretion) — no horizontal viewport overflow permitted |
| All breakpoints | Interactive tap targets (report links, accordion `<summary>`) minimum 44px height |
| SVG charts | Use `viewBox` + `width: 100%` so charts reflow without horizontal scroll at 375px |
