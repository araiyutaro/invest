<objective>
Supply the history data to the renderer. Add a thin fail-soft `loadUrgencyHistory` loader (reads `data/urgency-history.json`, falls back to `{}` on missing/corrupt) to `report-data-loaders.ts`, add it to the `Promise.all` loader batch in `generate-report.ts`, and pass the result as the 4th argument to `generatePortfolioReportHtml`. Anchor date is the already-in-scope `meetingResult.date`.

Purpose: Close the data path end-to-end (D-13) while guaranteeing that a missing or corrupt history file (expected on first run) never breaks report generation or deploy (D-14).
Output: Updated `report-data-loaders.ts` + `report-data-loaders.test.ts` + `generate-report.ts`.

Severity: `console.warn` (missing file is an expected first-run condition, matching `loadPrevPortfolioAnalysis`, not `console.error`).
</objective>
